# Plan: GitHub Stars

## Context
Fetch and store GitHub star counts per language. Requires storing one GitHub repo URL per language, then periodically fetching star counts via the GitHub API.

## Storing URLs
store language-dirname with full github URL in a TSV - this is a human generated file

## Storing Stars
**`stars.json`**
```json
{ "go": 125000, "python3": 63000 }
```

## pre-stars / stars Pattern
`pre-stars` generates `stars.sh` with one `curl` per repo for review:
```
curl -sf https://api.github.com/repos/golang/go
curl -sf https://api.github.com/repos/python/cpython
```
`stars` executes each line, parses the JSON response in Go (`json.Unmarshal`), extracts `stargazers_count`, saves `stars.jsonl` in each row.

Note: unauthenticated GitHub API is rate-limited to 60 req/hr. For >60 languages, a `GITHUB_TOKEN` env var should be read and passed as `-H "Authorization: Bearer $token"` in the curl header. `pre-stars` should inject this if the env var is set.

## Changes

### 1. Create `github-urls.tsv` (imported from spreadsheet)
Hand-populated or imported. Keyed by image tag.

### 2. Create `internal/tasks/pre-stars.go`
`PreStars()`:
- Reads `github-urls.tsv` into `map[string]string`.
- For each entry generates: `curl -sf https://api.github.com/repos/<owner/repo>`
- appends `-H "Authorization: Bearer <token>"`.
- Saves to `stars.sh` via `saveScript`.

### 3. Create `internal/tasks/stars.go`
`Stars()`:
- Reads and executes `stars.sh` line by line.
- Parses each JSON response, extracts `stargazers_count`.
- Saves rows as we fetch them to `stars.jsonl`.

### 4. Register in `cmd/app/main.go`
Add `pre-stars` and `stars` cases.
