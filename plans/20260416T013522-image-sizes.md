# Plan: Image Sizes

## Context
`internal/tasks/WIP-image-size.go` exists but reads from a hand-maintained `images.tsv` and has a `main()` func — it won't compile as part of the `tasks` package. Goal: replace it with `pre-weigh` / `weigh` commands that follow the same pattern as `pre-run` / `run`, using `discoverBuilds()` for image tags and outputting `image-sizes.json`.

## Hashmap Approach
Use `discoverBuilds()` to get all tags — no separate lookup needed. The tag is the key. Output: `map[string]string` (tag → human-readable size, e.g. `"bday/go": "45.2MB"`). A JSON object is more useful than an array for lookups.

```json
{ "bday/go": "45.2MB", "bday/python3": "128MB" }
```

## Changes

### 1. Delete `internal/tasks/WIP-image-size.go`

### 2. Create `internal/tasks/pre-weigh.go`
`PreWeigh()`:
- Calls `discoverBuilds(solutionsDir)` for tags.
- Generates `weigh.sh` with one line per tag: `docker image inspect <tag> --format {{.Size}}`
- Uses `saveScript(weighScript, lines, false)`.

Use `docker image inspect --format {{.Size}}` (returns bytes as int) rather than `docker image ls --format {{.Size}}` (returns human string like `45.2MB`) — bytes allows sorting and comparison.

### 3. Create `internal/tasks/weigh.go`
`Weigh()`:
- Reads `weigh.sh`, executes each line.
- Parses stdout as an integer (bytes).
- Builds `map[string]int` of tag → bytes.
- Saves to `image-sizes.json`.

### 4. Register in `cmd/app/main.go`
Add `pre-weigh` and `weigh` cases.

### 5. Add constants to `pre-build.go` (or a shared `constants.go`)
`weighScript = "weigh.sh"`, `imageSizesFile = "image-sizes.json"`
